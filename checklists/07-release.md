# リリース チェックリスト（詳細版）

## 1) Runbook/SOP バンドル

- 目的: 運用・保守作業の標準化
- 手順: 監視/アラート/WAF/DNS/SSL/Backup/Restore/ロールバックを文書化
- 成果物: Runbook/SOP
- 受入基準: 手順が実施可能レベルの具体性

## 2) 監視ダッシュボード

- 目的: 運用時に状態を見える化
- 手順: 主要指標/URL/担当/頻度の定義
- 成果物: ダッシュボード URL 一覧

## 3) アラート閾値/エスカレーション

- 目的: 障害の早期発見/迅速対応
- 手順: CPU/メモリ/レイテンシ/エラーレートの閾値設定、通知ルート定義
- 成果物: 閾値表

## 4) WAF 設定/ルール/テスト（必須）

- 目的: 攻撃遮断/誤検知低減
- 手順: ルール適用 → 検証 → 誤検知対応（除外/調整）
- 成果物: WAF 設定記録/テスト結果

## 5) DNS/SSL（有効期限管理）

- 目的: 接続安全性継続
- 手順: 証明書期限管理/更新手順
- 成果物: 期限表/更新記録

## 6) バックアップ/リストア検証

- 目的: 障害時の復旧確実化
- 手順: バックアップ取得 → リストア演習 → 記録
- 成果物: 検証レポート

## 7) ロールバック手順/判定基準

- 目的: リスク低減
- 手順: 判定条件/戻し手順/演習
- 成果物: ロールバック記録

## 8) リリース判定議事録/リリースノート/既知問題

- 目的: 決定の透明性/通知
- 手順: 判定会議 → 議事録 → ノート配布
- 成果物: 議事録/ノート
