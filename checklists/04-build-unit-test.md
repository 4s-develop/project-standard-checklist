# 構築・単体テスト チェックリスト（詳細版）

## 1) 単体テスト計画/仕様/結果

- 目的: モジュール単位の品質担保
- 手順: テスト対象/観点/データ/期待値を仕様化 → 実施 → 結果記録
- ツール: Jest/Playwright 等
- 成果物: 試験仕様/結果報告
- 受入基準: 主要ロジックの UT/IT/E2E が実行、重大欠陥なし
- 証跡: レポート + PR リンク

## 2) CI ゲート: Lint/SAST/SCA（必須）

- 目的: コード規約/脆弱性/依存問題の早期検出
- 手順: CI で自動実行（ESLint/Prettier/npm audit など）
- 成果物: CI ログ
- 受入基準: 失敗無し（警告はレビュー）
- 証跡: CI job links

## 3) カバレッジ（オプション）

- 目的: 将来必要時に指標化
- 手順: coverage 出力のみ。ゲート化はしない
- 証跡: lcov/レポート

## 4) 証跡保管ルール

- 目的: ログ/スクショの散逸防止
- 手順: SharePoint 既定フォルダへ格納 →PR にリンク
